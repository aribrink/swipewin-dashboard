Getting started

Tips

  • Do not start from scratch, use an existing asset and modify it to learn how it works.
  • Explore the sources for ideas and sample code.
  • Use Firebug or Chrome Developer Tools to find bugs on your website. Using one of those tools will help you to save time analyzing the site and finding elements structure, like classes, id or tags
  • Quick tip: open your site with Chrome, press F12 and go to console tab, reload your page and if something goes wrong you will see your page errors in red text.
  • In case of error messages, someone might have seen it too, so you can try a Google search for a quick fix.
 

Starting the app

To run this version you need to have installed Visual Studio 2017. After that, open the file mvc5-angular/Angle/Angle.sln. Once VS has been started, press F5 and it should build the project and open your default browser with the theme.

Main folders

Content/app

This folder contains the css files used for the theme. It contains the basic to display the layout and a few features. 

i18n

This folder contains the translation JSON files that uses the translate module to load string replacement for each language configured.

Images

Contains images used in the theme. 

Scripts/app

Contains the modules Javascript used in the theme (controllers, directive, routes, etc).
All this files are included in the app using a script bundle.

Sidebar

This folder contians the sidebar-menu.json. You will use this file to configure the sidebar menu items according to the states in the route configuration.

Vendor

We place here all vendor assets downdloaded manually. Some of them could be downloaded via NuGet but not all the package required are present in such system.  All this files are included in the app using bundles.
If you want to automate this folder, we suggest to use bower.

Views

Contains the views like any other MVC5 projects. But we mention this because some views needs the Layout = null; specified in order to retrieve only its content when AngularJS requires them.
 

Unlike the original AngularJS project, when using ng-include you request actions with their respective Views and Controller instead of single html files.

Layout

The main layout loads the necessary bundles to start the application and define a standard HTML document.

Angle/Views/Shared/_Layout.cshtml

This is the main layout

<!DOCTYPE html>
<html lang="en" data-ng-app="angle">

<head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1">
    <meta name="description" content="{{app.description}}">
    <meta name="keywords" content="app, responsive, angular, bootstrap, dashboard, admin">
    <title data-ng-bind="pageTitle()">Angle - Angular Bootstrap Admin Template</title>

    @Styles.Render("~/bundles/bootstrapStyles")
    @Styles.Render("~/bundles/appStyles")

    <!-- Themes-->
    <link rel="stylesheet" ng-href="{{app.layout.theme}}" data-ng-if="app.layout.theme">

</head>

<body data-ng-class="{ 'layout-fixed' : app.layout.isFixed, 
                       'aside-collapsed' : app.layout.isCollapsed, 
                       'aside-collapsed-text' : app.layout.isCollapsedText, 
                       'layout-boxed' : app.layout.isBoxed,
                       'layout-fs': app.useFullLayout, 
                       'hidden-footer': app.hiddenFooter, 
                       'layout-h': app.layout.horizontal, 
                       'aside-float': app.layout.isFloat, 
                       'offsidebar-open': app.offsidebarOpen, 
                       'aside-toggled': app.asideToggled }">

    <!-- Preloader -->
    <div data-preloader></div>
    <!-- App Content -->
    <div data-ui-view="main" data-autoscroll="false" class="wrapper">@RenderBody()</div>

    @Scripts.Render("~/bundles/baseScripts")
    @Scripts.Render("~/bundles/appScripts")

</body>
</html>
 
The @RenderBody() function will place the following markup from Views/App/Index.cshtml
<!-- top navbar-->
<header ng-include="'/Partials/TopNavbar'" class="topnavbar-wrapper"></header>
<!-- sidebar-->
<aside ng-include="'/Partials/Sidebar'" ng-controller="SidebarController" class="aside"></aside>
<!-- offsidebar-->
<aside ng-include="'/Partials/Offsidebar'" class="offsidebar hide"></aside>
<!-- Main section-->
<section ui-view="content">
    <!-- Page content-->
</section>
<!-- Page footer-->
<footer ng-include="'/Partials/Footer'"></footer>

Router

As you can note, the layout uses named ui-views in order to attach each view for the application.

The following code shows how are configured the important route states (from uiRouter) for the app views and the page views respectively

App route state

$stateProvider
    .state('app', {
        abstract: true,
        resolve: helper.resolveFor(...),
        views: {
            'content': {
                template: '<div data-ui-view="" autoscroll="false" ng-class="app.viewAnimation" class="content-wrapper"></div>',
                controller: ['$rootScope', function($rootScope) {
                    ...
                }]
            }
        }
    })

Pages route state

$stateProvider
    .state('page', {
        url: '/page',
        abstract: true,
        views: {
            'main': {
                templateUrl: helper.basepath('Pages/Page'),
                controller: ['$rootScope', function($rootScope) {
                    $rootScope.app.layout.isBoxed = false;
                }]
            }
        },
        resolve: helper.resolveFor('modernizr', 'icons')
    })

 

Usage

Adding new features (example)

This example shows how to add Toastr notifications to the seed project.
Once you have opened the project in your Visual Studio 2017 follow this steps:

1- Add a new controller 

 1.a- In the Solution explorer, right click on folder Controllers and select Add -> Controller. Name it NotificationsController

2- Now we will add a new view that will contains the markup for Toaster. 

 2.a - Step 1.a must have created a folder Views/Notifications (if not, add it manually)

 2.b - Right click on the new folder and select Add -> View and name it Index

 2.c - Paste the following markup into the new file (short example from file notifications.html in angularjs project)

@{
    Layout = null;
}
<h3>Notifications</h3>
<div class="panel panel-default">
    <div class="panel-heading">Toaster notifications for Angular</div>
    <div ng-controller="ToasterDemoCtrl" class="panel-body">
        <toaster-container toaster-options="{'position-class': 'toast-top-right', 'close-button':true}"></toaster-container>
        <div class="row">
            <div class="col-sm-6">
                <div class="form-group">
                    <label>Title</label>
                    <input type="text" placeholder="Enter a title..." ng-model="toaster.title" class="form-control" />
                </div>
                <div class="form-group">
                    <label>Message</label>
                    <textarea placeholder="Enter a message" rows="5" ng-model="toaster.text" class="form-control"></textarea>
                </div>
            </div>
            <div class="col-sm-6">
                <div class="row">
                    <div class="col-md-6">
                        <p>Toast Type</p>
                        <div>
                           <input type="radio" name="toasts" ng-model="toaster.type" value="success" />
                           Success
                        </div>
                        <div>
                           <input type="radio" name="toasts" ng-model="toaster.type" value="info" />
                           Info
                        </div>
                        <div>
                           <input type="radio" name="toasts" ng-model="toaster.type" value="wait" />
                           Wait
                        </div>
                        <div>
                           <input type="radio" name="toasts" ng-model="toaster.type" value="warning" />
                           Warning
                        </div>
                        <div>
                           <input type="radio" name="toasts" ng-model="toaster.type" value="error" />
                           Error
                        </div>
                    </div>
                </div>
            </div>
        </div>
        <button ng-click="pop()" class="btn btn-info">Show a Toaster</button>
    </div>
</div>

3- Now we will add the new Sidebar entry

 3.a - In the Solution explorer, click on file Sidebar/sidebar-menu.json and add the following at the end

  {
    "text": "Notifications",
    "sref": "app.notifications",
    "icon": "fa fa-file-o"
  }

4- We wiil add a route for the new view

 4.a - In the Solution explorer, edit the file Scripts/app/modules/routes/routes.config.js and add the following route state

    .state('app.notifications', {
        url: '/Notifications',
        title: 'Notifications',
        templateUrl: helper.basepath('Notifications'),
        resolve: helper.resolveFor('toaster')
    })
 
5- Now we have to add the Toaster assets and the AngularJS controller
 
  5.a Copy folder /vendor/angularjs-toaster and paste into your project Vendor folder
 
  5.b Copy the file  /master/js/modules/controllers/demo-toaster.js and paste into your project Scripts/app/modules/controllers folder
 
  5.c In solution explorer, edit file Scripts/app/modules/lazyload/lazyload.constants.js and uncomment the line
  { name: 'toaster', files: ['/Vendor/angularjs-toaster/toaster.js','/Vendor/angularjs-toaster/toaster.css'] }

  

6- Done. Now you can run your solution and the new view with Toaster notifications should be ready.

 

Seed Project

This project is an application skeleton. You can use it to quickly bootstrap your MVC5 webapp projects and dev environment for these projects. 
The seed app doesn't do much and has most of the feature removed so you can add theme as per your needs just following the demo app examples.
 
The angular seed project includes the following features
 
- Modernizr
- Icons (FontAwesome and SimpleLine)
- Translation
- RTL mode
- Top navbar search form
- Themes
- Two views with its own sidebar entries (one single item and a menu)
- Empty offsidebar (left sidebar)
