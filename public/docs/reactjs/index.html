Getting started

Tips

  • Do not start from scratch, use an existing asset and modify it to learn how it works.
  • Explore the sources for ideas and sample code.
  • Use Firebug or Chrome Developer Tools to find bugs on your website. Using one of those tools will help you to save time analyzing the site and finding elements structure, like classes, id or tags
  • Quick tip: open your site with Chrome, press F12 and go to console tab, reload your page and if something goes wrong you will see your page errors in red text.
  • In case of error messages, someone might have seen it too, so you can try a Google search for a quick fix.
 

Starting the app

Since this is a ReactJS based application you need a server (Apache, IIS, xampp, node, etc) to serve the html files and perform http request to load all views.
 
Important! Opening the index.html with a double click (i.e. using file:// protocol) will show you only a blank page because there’s no server that response to the requests made for each view in order to display the app interface.

Webpack development server

See section Build to install node and dependecies and then run webpack task to automatically serve the app

Main folders explanation

 
dist/ folder
 
This folder contains the compiled files. This files are ready to deploy on your server.
 
app/ folder
 
This folder contains the source files. You will find the following folders inside
 
styles/ This folder contains the SCSS files for the core styles and bootstrap styles.
app/ Here you will find JSX files for React components.
img/ Images used in the copy (only copied to dist folder)
fonts/ Fontsused in the copy (only copied to dist folder)
server/ Contains static json files used to emulate server calls
 

Custom code

JSX: Add a new component folder to create new ReactJS components and import them into the file App.jsx, then follow the code examples to associate components to routes.
 
SASS: create a custom folder inside the sass folder and @import your files from app.scss
 
 
 
The premise is, the less you change the downloaded code, the easier will be to apply any updates. Try always to keep your own code the most separated as possible from the package code to easily apply new updates when necessary.

 

 

Build

Node.js is a platform built on Chrome’s JavaScript runtime for easily building fast, scalable network applications.
 
Webpack is a module bundler. This means webpack takes modules with dependencies and emits static assets representing those modules.
 
Installing tools
 
The following steps are intended to be an orientation guide, if you are not experienced with this you will need to learn more about it from Google :)
 
  • To install node and npm, go to http://nodejs.org/
  • Download and install GIT for your platform http://git-scm.com/downloads
 
Once you have all tools installed
 
  • Open a terminal, in the root folder of the project, run the command npm install. This command will install all project dependencies.

Main commands

This command will start a server in development mode
$ npm run start
This command will compile all files for distribution (dist folder)
$ npm run build

Base Href

If you need to set a base href for your project before compilation, set the environment variable WP_BASE_HREF and then run compilation.

Example:

$ set WP_BASE_HREF=/some/web/path
$ npm run build

Webpack and React Hot Loader

The application is built using WebPack which is a tool that analyze the code and builds a script ready for web browsers.

The configuration of webpack is included in files:

webpack.common.js: common setup for development and production
webpack.dev.js: setup for development mode (hot-loader server)
webpack.prod.js: setup for production mode

react-hot-loader allows to reload components easily without waiting to recompile the entire app.

 

Usage

Layout

Layout can be changed via the following classed applied to the body tag
 
.layout-fixed: Makes navbars become fixed while the user can scroll only content
 
.layout-boxed: Limits the width of the main wrapper element
 
.aside-collapsed: Condenses the sidebar showing only icons
 
.aside-collapsed-text: Condenses the sidebar showing icons and Text
 
.aside-toggle: used internally for mobiles to hide the sidebar off screen
 
.offsidebar-open: used internally to display the offsidebar component (formally the right sidebar)
 

JSX

The Javascript (JSX components) are bundled into one single that contains all react components declared under the components folder
 
The application starts using the file App.jsx. This import all components and declare all routes associating components to each route.
 
In Vendor.jsx you will find all imported third party dependencies, you can add more or remove anything here according to your project.

Routes

 
To declare route we use react-router which is basically a routing library based on React components
You can note that according to the layout you are using is necessary to wrap each route with a Base* layout component. Example
 
ReactDOM.render(
    <Router history={browserHistory}>
        <Route path="/" component={Base}>

            {/* Default route*/}
            <IndexRoute component={DashboardV1} />

            ...

        </Route>

        {/*Horizontal Layout*/}
        <Route path="/" component={BaseHorizontal}>
            ...
        </Route>

        {/*Pages*/}
        <Route path="/" component={BasePage}>
            ...
        </Route>

        {/* Not found handler */}
        <Route path="*" component={NotFound}/>

    </Router>,
    document.getElementById('app')
);
 

Bootstrap components

 
To make the application the most ReactJS-friendly we have included react-bootstrap which is basically the popular framework rebuilt for React

Writing components

 
Each component is declared using the following basic code using ES6
import React from 'react';
import ContentWrapper from '../Layout/ContentWrapper';

class MyComponent extends React.Component {

    render() {
        return (
            <ContentWrapper>
                ...
            </ContentWrapper>
            );
    }

}

export default MyComponent;
 
ContentWrapper is a component that must be used to wrap each view in the app.
 
Then from the App.jsx you can import components with
import MyComponent from './path/to/MyComponent';
 
To keep the Javascript that controls the behavior of plugins separated from the associated component we use the following convention to name files (optional)
 
MyComponent.run.js
 
As you can see, we are just adding the "run" word and the extension is JS
Inside you can declare a function to be exported
 
export default function() {
    // code..
}
 
Then from the component JSX you can import this file
 
import MyComponenttRun from './MyComponentt.run';
 
And using the componentDidMount() call the imported function
 
componentDidMount() {
   MyComponenttRun();
}
 

SASS

SASS files are used to compile the main application styles into the file app.css and the custom Bootstrap styles into bootstrap.css
 

RTL

RTL is based on rtlcss plugin using a custom loader that allows to change the application styles into an RTL representation.
To enable RTL open the file webpack.common.js and uncommment the line that uses the rtlcss-loader
 
{
   loader: 'rtlcss-loader' // uncomment for RTL
}

 

 

 

Themes

To change the color scheme:

From SASS files

 
Edit the SASS files in folder app/styles/app and the file app/styles/bootstrap/variables.scss to use the color you want.
Like Bootstrap, most of the colors are based on $brand-* variables.
 
You can also edit the files in app/styles/themes folder to create your own set of color schemes. 
See the file app/components/Common/load-themes.js to lear how theme are imported and included into the page.
 
Changing the theme colors from SASS files helps you to avoid bloating your css by not double declaring your color rules.
 

Seed Project

This project is an application skeleton. You can use it to quickly bootstrap your ReactJS webapp projects and dev environment for these projects. 
The seed app doesn't do much and has most of the feature removed so you can add theme as per your needs just following the demo app examples.
 
This project is provided in order to start with the template using a different approach. Usually, templates will come with all features working and you need to remove them one by one in the way you don't need them.
 
With the seed project you can start adding custom features and others from the full project to make grow your app.
 
Since the files and structure is the same for the full and seed projects, you can save time using comparison tools that allows to apply changes from full features project into the seed project.
 
 

 

Heroku

In first place we have to create the following files used by Heroku to run a server. Place this new files in the root folder of the project

Procfile

web: node server.js
server.js
var express = require('express');
var app = express();
var path = require('path');

app.use(express.static(path.resolve(__dirname, 'dist')));

app.get('*', (req, res) => {
    res.sendFile(path.resolve(__dirname, 'dist', 'index.html'));
});

app.listen(process.env.PORT || 9090);
 
Follow this steps to deploy this application to heroku:
 
  1. Run heroku login to login on heroku with your credentials
  2. Start a new repot with git init
  3. Create your application with heroku create app-name
  4. Add files for commit with git add .
  5. Commit files git commit -m "initial commit"
  6. Run heroku config:set NPM_CONFIG_PRODUCTION=false
  7. Rename in package.json script named: "postinstall-heroku" to "postinstall"
  8. Upload to heroku: git push heroku master
  9. If everything went fine, open in browser with heroku open

 

 

Refactor

This section provides a brief description of the new structure. The main change is that we have removed the entire Gulp and BrowserSync workflow in favor of using only Webpack which is a more used and suitable build tool for a React development.

Folders

'master' has been renamed to 'app'

'jsx' folder has been removed and all component resides under 'components'

'sass' folder has been renamed to 'styles'

The new folder 'fonts' is used to store all font files used in the project

Vendor

The new file Vendor.jsx is used now replacing the old plain 'vendor.json'
Using this file we can import all third party assets into the app allowing a more flexible inclusion of vendor assets.

Themes

Themes are used in a different way, instead of requesting the stylesheets with the removed load-css script, they are imported into the main bundle and injected into the app by creating a style tag into the head section. The new file that handles this behavior is /components/Common/load-themes.js

Other minor changes

Separated from the purpose of this little refactor, we have improved things and fixed other minor bugs, like the correct use of Link tags, updated react-router to 3.0.0, fixed a bug with the routes reload and other minor changes.

Since there's no automatic process to upgrade this product, we suggest you to use any comparison tool like WinMerge, BeyondCompare or Kaleidoscope. By comparing the folder your are working on against the new folders you can see the difference and implement new changes.
