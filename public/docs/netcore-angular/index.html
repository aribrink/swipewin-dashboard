  • In case of error messages, someone might have seen it too, so you can try a Google search for a quick fix.

Starting the app

 
To run this version you need to have installed Visual Studio 2017 and ASP5 from https://get.asp.net/
 
You will need also a nodejs enviroment, gulp and bower
 
 
Node.js is a platform built on Chrome’s JavaScript runtime for easily building fast, scalable network applications.
 
Gulp is a task manager, you can define different tasks to run certain commands. Those commands does the compilation job for JADE and LESS, and concatenates the JS files.
 
Bower is a dependency manager, it works by fetching and installing packages from all over, taking care of hunting, finding, downloading, and saving the stuff you’re looking for. Bower keeps track of these packages in a manifest file, bower.json.
 
 
Installing tools
 
The following steps are intended to be an orientation guide, if you are not experienced with this you will need to learn more about it from Google :)
 
  • To install node and npm, go to http://nodejs.org/
  • Run npm install -g bower to install bower to manage dependencies
  • Download and install GIT for your platform http://git-scm.com/downloads
 
Once you have all tools installed
 
  • Open a terminal, go the folder netcore-angular/Angle/Angle
  • Run npm install. This command will install gulp and all project dependencies.
  • Run bower install to install vendor dependencies
  • Finally run gulp to build assets
 
 
After that, open the file netcore-angular/Angle/Angle.sln.
 
Once VS has been started, press F5 and it should build the project and open your default browser with the theme.
 
We have binded the gulp task to the build process of VS so when you build your app, VS should run gulp default task to generated and put up to date all assets.
 
Issues
 
If you found problems while running gulp, please check that you are using the last node version in your system here

Router

As you can note, the layout uses named ui-views in order to attach each view for the application.

The following code shows how are configured the important route states (from uiRouter) for the app views and the page views respectively

App route state

$stateProvider
    .state('app', {
        abstract: true,
        resolve: helper.resolveFor(...),
        views: {
            'content': {
                template: '<div data-ui-view="" autoscroll="false" ng-class="app.viewAnimation" class="content-wrapper"></div>',
                controller: ['$rootScope', function($rootScope) {
                    ...
                }]
            }
        }
    })

Pages route state

$stateProvider
    .state('page', {
        url: '/page',
        abstract: true,
        views: {
            'main': {
                templateUrl: helper.basepath('Pages/Page'),
                controller: ['$rootScope', function($rootScope) {
                    $rootScope.app.layout.isBoxed = false;
                }]
            }
        },
        resolve: helper.resolveFor('modernizr', 'icons')
    })

 

Usage

Adding new features (example)

This example shows how to add Toastr notifications to the seed project.
Once you have opened the project in your Visual Studio 2017 follow this steps:

1- Add a new Controller 

 1.a- In the Solution explorer, right click on folder Controllers and select Add -> Controller. Name it NotificationsController

2- Now we will add a new view that will contains the markup for Toaster. 

 2.a - Step 1.a must have created a folder Views/Notifications (if not, add it manually)

 2.b - Right click on the new folder and select Add -> View and name it Index

 2.c - Paste the following markup into the new file (short example from file notifications.html in angularjs project)

@{
    Layout = null;
}
<h3>Notifications</h3>
<div class="panel panel-default">
    <div class="panel-heading">Toaster notifications for Angular</div>
    <div ng-controller="ToasterDemoCtrl" class="panel-body">
        <toaster-container toaster-options="{'position-class': 'toast-top-right', 'close-button':true}"></toaster-container>
        <div class="row">
            <div class="col-sm-6">
                <div class="form-group">
                    <label>Title</label>
                    <input type="text" placeholder="Enter a title..." ng-model="toaster.title" class="form-control" />
                </div>
                <div class="form-group">
                    <label>Message</label>
                    <textarea placeholder="Enter a message" rows="5" ng-model="toaster.text" class="form-control"></textarea>
                </div>
            </div>
            <div class="col-sm-6">
                <div class="row">
                    <div class="col-md-6">
                        <p>Toast Type</p>
                        <div>
                           <input type="radio" name="toasts" ng-model="toaster.type" value="success" />
                           Success
                        </div>
                        <div>
                           <input type="radio" name="toasts" ng-model="toaster.type" value="info" />
                           Info
                        </div>
                        <div>
                           <input type="radio" name="toasts" ng-model="toaster.type" value="wait" />
                           Wait
                        </div>
                        <div>
                           <input type="radio" name="toasts" ng-model="toaster.type" value="warning" />
                           Warning
                        </div>
                        <div>
                           <input type="radio" name="toasts" ng-model="toaster.type" value="error" />
                           Error
                        </div>
                    </div>
                </div>
            </div>
        </div>
        <button ng-click="pop()" class="btn btn-info">Show a Toaster</button>
    </div>
</div>

3- Now we will add the new Sidebar entry

 3.a - In the Solution explorer, click on file wwwroot/server/sidebar-menu.json and add the following at the end.

  {
    "text": "Notifications",
    "sref": "app.notifications",
    "icon": "fa fa-file-o"
  }

4- We wiil add a route for the new view

 4.a - In the Solution explorer, edit the file wwwroot/master/js//modules/routes/routes.config.js and add the following route state

    .state('app.notifications', {
        url: '/Notifications',
        title: 'Notifications',
        templateUrl: helper.basepath('Notifications'),
        resolve: helper.resolveFor('toaster')
    })
 
5- Now we have to add the Toaster assets and controller
 
  5.a In solution explorer, Right Click on project name and select "Manage Bower Packages"
 
  5.b Search for angularjs-toaster and click Install button
 
  5.c Copy from the full project the file wwwroot/master/js/modules/elements/toaster.controller.js. Add this file also into the file Views/Shared/_Layout.html
 
  5.d In solution explorer, edit file wwwroot/master/js/modules/lazyload/lazyload.constants.js and uncomment the line
  { name: 'toaster', files: ['/Vendor/angularjs-toaster/toaster.js','/Vendor/angularjs-toaster/toaster.css'] }

  

6- Done. Now you can run your solution and the new view with Toaster notifications should be working.

Seed Project

This project is an application skeleton. You can use it to quickly bootstrap your NetCore webapp projects and dev environment for these projects. 
The seed app doesn't do much and has most of the feature removed so you can add theme as per your needs just following the demo app examples.
 
The seed project includes the following features
 
- Modernizr
- Icons (FontAwesome and SimpleLine)
- Translation
- RTL mode
- Top navbar search form
- Themes
- Two views with its own sidebar entries (one single item and a menu)
- Empty offsidebar (left sidebar)
